Welcome!

Nakadi-java is a client for the Nakadi Event Broker API.

  • Straightforward. Easy to use as is, or as an engine for higher level abstractions.

  • Complete. A full reference implementation of the Nakadi API for producers and consumers.

  • Robust. HTTP requests and consumer streams recover from common connection and server issues.

  • Minimal. No heavyweight dependencies, the sole dependency is on SLF4J.

  • Operable. Errors, logging and instrumentation matter as much as functionality.

Using the Client

Creating a client is as simple as: 

NakadiClient client = NakadiClient.newBuilder()
  .baseURI("http://localhost:9080")
  .build();

With that you can post type safe events:

DataChangeEvent<PriorityRequisition> dce23 = new DataChangeEvent<PriorityRequisition>()
  .metadata(new EventMetadata())
  .op(DataChangeEvent.Op.C)
  .data(new PriorityRequisition("23"));

DataChangeEvent<PriorityRequisition> dce24 = new DataChangeEvent<PriorityRequisition>()
  .metadata(new EventMetadata())
  .op(DataChangeEvent.Op.C)
  .data(new PriorityRequisition("24"));

Response batch = resource.send("priority-requisitions", dce23, dce24);

And consume events from the stream with just a few lines of code:

StreamConfiguration configuration = new StreamConfiguration()
    .subscriptionId("27302800-bc68-4026-a9ff-8d89372f8473");

StreamProcessor processor = client.resources().streamBuilder(configuration)
    .streamObserverFactory(new LoggingStreamObserverProvider())
    .build();

processor.start();
