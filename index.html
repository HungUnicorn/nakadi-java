Welcome!

Nakadi-java provides an implementation of the Nakadi Event Broker API.

  • Straightforward. An easy to use client suitable for direct use, or as an engine for higher level abstractions.

  • Complete. A full reference implementation of the Nakadi API for producers and consumers.

  • Robust. HTTP request/responses and consumer stream handling recovers from common connection and server problems.

  • Minimal Dependencies. No dependency on heavyweight frameworks or libraries, the sole dependency is on the SLF4J API.

  • Operations Friendly. Error handling, logging and instrumentation are given the same importance as functionality.

Using the Client

Creating a client is as simple as: 

NakadiClient client = NakadiClient.newBuilder()
  .baseURI("http://localhost:9080")
  .build();

With that you can post type safe events:

DataChangeEvent<PriorityRequisition> dce1 = new DataChangeEvent<PriorityRequisition>()
  .metadata(new EventMetadata())
  .op(DataChangeEvent.Op.C)
  .data(new PriorityRequisition("24"));

DataChangeEvent<PriorityRequisition> dce2 = new DataChangeEvent<PriorityRequisition>()
  .metadata(new EventMetadata())
  .op(DataChangeEvent.Op.C)
  .data(new PriorityRequisition("24"));

Response batch = resource.send("priority-requisitions", dce1, dce2);

And consume events from the stream with just a few lines of code:

StreamConfiguration configuration = new StreamConfiguration()
    .subscriptionId("27302800-bc68-4026-a9ff-8d89372f8473");

StreamProcessor processor = client.resources().streamBuilder(configuration)
    .streamObserverFactory(new LoggingStreamObserverProvider())
    .build();

processor.start();
